# 프로젝트 개요 (쉬운 설명)

## 1주차: 팩터 투자 (Python)

**뭘 한 거야?**

"어떤 주식을 사면 돈을 벌까?"에 대한 답을 찾았어.

연구 결과, 특정 특징을 가진 주식들이 평균적으로 수익이 더 좋더라:
- **모멘텀**: 최근 1년간 많이 오른 주식 → 앞으로도 오르는 경향
- **가치**: 회사 가치 대비 주가가 싼 주식 → 장기적으로 수익 좋음
- **퀄리티**: 돈을 잘 버는 회사 → 안정적인 수익

이걸 **10년치 한국 주식 데이터**로 직접 테스트해봤어. 결과: 모멘텀 전략으로 연 40% 수익!

## 2주차: 포트폴리오 최적화 (Rust)

**뭘 한 거야?**

"좋은 주식들을 찾았는데, 각각 얼마씩 사야 해?"에 대한 답을 찾았어.

**문제**: 삼성전자, SK하이닉스, 네이버에 투자하고 싶은데...
- 삼성 50%, SK 30%, 네이버 20%?
- 아니면 삼성 80%, 나머지 10%씩?

**해결책 1: Mean-Variance (마코위츠)**

1952년에 나온 방법이야. 핵심 아이디어:

> "계란을 한 바구니에 담지 마라"

수학적으로 **위험은 최소화**하면서 **수익은 최대화**하는 비중을 찾아줘.

**해결책 2: Black-Litterman**

마코위츠의 업그레이드 버전이야.

마코위츠 문제점: "미래 수익률을 어떻게 알아?" → 모르니까 결과가 이상해짐

Black-Litterman 해결법:
1. 일단 **시장을 믿어** (현재 시장 가격이 "정답"이라고 가정)
2. 거기에 **내 의견을 섞어** ("삼성전자 오를 것 같아" 같은)
3. 확신이 있으면 많이, 없으면 조금만 반영

---

## 왜 Rust로 했어?

Python은 느려. 주식 100개의 최적 비중 찾으려면 행렬 계산을 엄청 많이 해야 해.

Rust는 C/C++만큼 빠르면서 안전해. 그래서:
- 빠른 계산 → Rust
- 편한 사용 → Python에서 호출

**결과**: Python에서 `from quant_core import ...` 하면 Rust로 만든 빠른 코드 사용 가능!

## 프로젝트 구조

```
quant/
├── src/                    # 1주차: 팩터 투자 (Python)
│   ├── factors/            # 모멘텀, 가치, 퀄리티 계산
│   ├── portfolio/          # 백테스트
│   └── analytics/          # 성과 분석
│
├── quant-core/             # 2주차: 포트폴리오 최적화 (Rust)
│   └── src/
│       ├── optimizer/      # 마코위츠, Black-Litterman
│       └── risk/           # 위험 계산
│
└── docs/                   # 설명서
    ├── overview.md         # 이 파일 (쉬운 설명)
    ├── factor_investing.md # 팩터 투자 상세
    └── portfolio_optimization.md  # 포트폴리오 최적화 상세
```
