# 문제 1: 아메리칸 옵션

## 무엇을 계산하나?

**아메리칸 풋옵션**의 현재 가격을 계산합니다.

### 옵션이란?

**콜옵션 (Call)**: 주식을 정해진 가격(K)에 **살 권리**
```
주가 = 110원, 행사가 K = 100원
→ 100원에 사서 110원에 팔기 → 이익 10원
```

**풋옵션 (Put)**: 주식을 정해진 가격(K)에 **팔 권리**
```
주가 = 90원, 행사가 K = 100원
→ 90원에 사서 100원에 팔기 → 이익 10원
```

### 유럽형 vs 아메리칸

| 구분 | 유럽형 | 아메리칸 |
|-----|-------|---------|
| 행사 시점 | **만기**에만 | **언제든지** |
| 가격 | 낮음 | 높음 (더 유리) |
| 계산 | 쉬움 | 어려움 |

**예시**:
```
오늘: 주가 80원
1개월 후: 주가 90원
만기(3개월): 주가 105원
행사가 K = 100원 (풋옵션)

유럽형: 만기까지 기다림 → 손해 (105 > 100)
아메리칸: 1개월 후 조기행사 → 이익 10원! (100 - 90)
```

## 어떻게 계산하나?

### 1단계: 그리드 설정

주가 공간을 격자로 나눔:
```
주가 그리드: [0, 20, 40, 60, 80, 100, 120, ..., 400]
시간 그리드: [0, dt, 2dt, 3dt, ..., T(만기)]
```

### 2단계: 만기 페이오프

만기에서 옵션 가치는 확실히 알 수 있음:
```
풋옵션 페이오프 = max(K - S, 0)

K = 100일 때:
S = 80  → 100 - 80 = 20원
S = 100 → 100 - 100 = 0원
S = 120 → max(-20, 0) = 0원
```

### 3단계: 역방향 계산 (Backward Induction)

**시간을 거슬러 올라가며 계산**:

```
만기(t=T):     [20, 15, 10, 5, 0, 0, ...]  ← 페이오프
↓ 역산
t=T-dt:        [22, 17, 11, 4, 0, 0, ...]
↓ 역산
t=T-2dt:       [23, 18, 12, 5, 0, 0, ...]
↓ ...
현재(t=0):     [30, 25, 20, 15, 10, 5, ...] ← 최종 가격!
```

**왜 역방향?**
- 만기 가치는 확실히 알고 있음 (페이오프)
- 현재는 모름 (그래서 계산하는 것!)
- 미래에서 과거로 거슬러 올라가며 계산

### 4단계: PSOR (Projected SOR)

**PSOR = Projected Successive Over-Relaxation**

#### SOR이란?

선형방정식을 푸는 **반복적 방법**:
```python
# 방정식: Ax = b
# 초기 추측: x = [1, 1, 1, ...]

반복 1회: x = [1.2, 0.9, 1.1, ...]
반복 2회: x = [1.15, 0.95, 1.08, ...]
반복 3회: x = [1.14, 0.96, 1.07, ...]
...
수렴!
```

**omega (ω)**: 수렴 속도 조절
- ω = 1: 일반 Gauss-Seidel
- ω = 1.2: 과완화 → 더 빠르게 수렴

#### Projected란?

매 반복마다 **제약조건을 강제**:
```
제약조건: 옵션가치 ≥ 내재가치 (조기행사 가치)

계산된 값: v_temp = 8원
내재가치: payoff = 10원
→ v_new = max(8, 10) = 10원  ← 투영!
```

**왜 필요?**
- 아메리칸 옵션은 언제든지 조기행사 가능
- 따라서 옵션 가치 < 내재가치 일 수 없음
- 그러면 즉시 행사하는 게 이득!

## 핵심 공식

### Black-Scholes PDE (완전 해부)

```
∂V/∂t + (r-q)S∂V/∂S + 0.5σ²S²∂²V/∂S² - rV = 0
```

#### 각 항의 의미 (자세히)

**1. ∂V/∂t (시간 미분)**
```
옵션 가치가 시간에 따라 어떻게 변하는가?

예시:
  오늘: 옵션 가치 = 10원
  내일: 옵션 가치 = 9.9원
  → ∂V/∂t ≈ -0.1원/일

만기가 가까워질수록:
  - 시간 가치 감소 (Time Decay)
  - Theta와 관련
```

**2. (r-q)S∂V/∂S (드리프트항)**
```
주가가 평균적으로 올라가는 추세

r = 무위험 이자율 (예: 3% = 0.03)
q = 배당률 (예: 1% = 0.01)
r - q = 순 성장률 = 2%

∂V/∂S = 주가 변화에 대한 옵션 가치 변화 (Delta)

의미: "주가가 추세대로 움직일 때 옵션 가치는?"

예시:
  r = 0.03, q = 0.01, S = 100
  → 주가는 연 2% 성장 기대
  → Delta = 0.5라면
  → 이 항 = (0.03-0.01) × 100 × 0.5 = 1
```

**3. 0.5σ²S²∂²V/∂S² (변동성항)**
```
주가가 랜덤하게 흔들리는 효과

σ = 변동성 (예: 30% = 0.3)
σ² = 분산
∂²V/∂S² = 옵션 가치의 곡률 (Gamma)

의미: "주가가 위아래로 흔들릴 때 옵션 가치는?"

왜 0.5?
→ 확률론적 계산 (Ito's Lemma)에서 나온 상수
→ 기하 브라운 운동의 2차항

예시:
  σ = 0.3 (30% 변동성)
  S = 100
  Gamma = 0.02
  → 이 항 = 0.5 × (0.3)² × (100)² × 0.02 = 9
  → 변동성이 클수록 이 항이 큼
```

**4. -rV (할인항)**
```
돈의 시간 가치

r = 무위험 이자율 = 3%
V = 옵션 현재 가치 = 10원

의미: "현재 돈 vs 미래 돈"

예시:
  오늘 10원 = 1년 후 10 × e^(0.03) ≈ 10.30원
  → 미래 가치를 현재로 할인
  → 이 항 = -0.03 × 10 = -0.3
```

#### 왜 이 방정식이 중요한가?

**1. 무차익 조건 (No-Arbitrage)**
```
주식 + 옵션 포트폴리오 = 무위험 이자율

만약 이 방정식을 만족하지 않으면?
→ 차익거래 기회 발생!
→ 시장이 효율적이면 바로 사라짐
```

**2. 옵션 가격의 "법칙"**
```
뉴턴의 운동 법칙처럼
옵션 가격이 따라야 하는 자연 법칙

이 PDE를 풀면 → 옵션의 공정 가격
```

#### 직관적 이해

```
옵션 가치의 변화 = 시간 효과 + 주가 추세 + 주가 변동성 - 할인

∂V/∂t = -(r-q)S∂V/∂S - 0.5σ²S²∂²V/∂S² + rV
└─────┘   └──────────┘   └──────────────┘   └─┘
시간변화   드리프트       변동성            할인

균형 상태:
  시간 감소 = 주가 상승 기대 + 변동성 이득 - 할인 비용
```

#### 간단한 비유

**옵션 = 물 흐르기**
```
∂V/∂t: 시간이 지나면서 물이 빠짐 (증발)
(r-q)S∂V/∂S: 경사면을 따라 물이 흐름 (추세)
0.5σ²S²∂²V/∂S²: 물이 사방으로 퍼짐 (확산)
-rV: 중력으로 물이 아래로 (할인)

균형점: 이 모든 효과가 합쳐져서 0
```

### Crank-Nicolson 방법

**시간 이산화 방법**:
- 암시적(implicit) + 명시적(explicit)의 평균
- theta = 0.5
- 무조건 안정적 (큰 시간 스텝 가능)
- 2차 정확도

#### 명시적 vs 암시적 자세히

**명시적 (Explicit) 방법**:
```
다음 값 = 현재 알고 있는 값들로 계산

V[t+dt] = f(V[t], V[t-dt], ...)
         └─ 이미 알고 있음!

장점: 계산 쉬움 (바로 대입)
단점: 불안정할 수 있음 (dt를 작게 해야 함)
```

**암시적 (Implicit) 방법**:
```
현재 값 = 미래 값을 포함한 방정식 풀기

V[t] = f(V[t+dt], V[t], V[t-dt], ...)
      └─ 모름! 방정식 풀어야 함

장점: 안정적 (dt를 크게 해도 됨)
단점: 계산 복잡 (연립방정식 풀어야 함)
```

**Crank-Nicolson = 둘의 평균**:
```
(명시적 + 암시적) / 2

장점만 결합:
- 안정적 (암시적의 장점)
- 정확도 높음 (2차)
- 적당한 계산 비용
```

**예시**:
```python
# 명시적 (dt 작아야 안정)
V_new[i] = V[i] + dt * (...)

# 암시적 (방정식 풀기)
# (1 - 0.5*dt*A) * V_new = V

# Crank-Nicolson (중간)
# (1 - 0.5*dt*A) * V_new = (1 + 0.5*dt*A) * V
#  └─ 암시적 부분          └─ 명시적 부분
```

### 그릭스 (Greeks) - 완전 가이드

**옵션 민감도 지표**:

| 그릭 | 의미 | 예시 |
|-----|------|------|
| **Delta (Δ)** | 주가 1원 변화 → 옵션가 변화 | 0.5 → 주가 ↑1원 시 옵션가 ↑0.5원 |
| **Gamma (Γ)** | 주가 1원 변화 → 델타 변화 | 0.02 → 주가 ↑1원 시 델타 ↑0.02 |
| **Theta (Θ)** | 하루 지남 → 옵션가 변화 | -0.05 → 하루 후 옵션가 -0.05원 |

#### Delta (Δ) 자세히

**정의**: ∂V/∂S (주가에 대한 1차 미분)

**의미**:
```
주가가 1원 변할 때 옵션 가격은 몇 원 변하나?

콜옵션:
  Delta = 0.5
  주가 100 → 101 (↑1원)
  옵션가 10 → 10.5 (↑0.5원)

풋옵션:
  Delta = -0.5
  주가 100 → 101 (↑1원)
  옵션가 10 → 9.5 (↓0.5원, 음수!)
```

**헤징에 활용**:
```
옵션 1개 매도 = 주식 Delta개 매수

예: Delta = 0.5인 콜옵션 1개 매도
→ 주식 0.5주 매수
→ 주가 변동에 중립 (헤지 완료)
```

**범위**:
```
콜옵션: 0 ~ 1
  ITM (내가격): Delta ≈ 1
  ATM (등가격): Delta ≈ 0.5
  OTM (외가격): Delta ≈ 0

풋옵션: -1 ~ 0
  ITM: Delta ≈ -1
  ATM: Delta ≈ -0.5
  OTM: Delta ≈ 0
```

#### Gamma (Γ) 자세히

**정의**: ∂²V/∂S² (주가에 대한 2차 미분)

**의미**:
```
Delta가 얼마나 빨리 변하나?
= 옵션 가격 곡선의 "굽은 정도"

주가 변화 → Delta 변화

예:
  주가 100: Delta = 0.50, Gamma = 0.02
  주가 101: Delta = 0.50 + 0.02 = 0.52
  주가 102: Delta = 0.52 + 0.02 = 0.54
```

**왜 중요한가?**:
```
Gamma가 크면:
  - Delta가 빠르게 변함
  - 자주 재헤징 필요
  - 위험하지만 기회도 큼

Gamma가 작으면:
  - Delta가 천천히 변함
  - 헤징 안정적
  - 안전하지만 수익도 적음
```

**ATM 옵션이 Gamma 최대**:
```
  Gamma
    ↑
    │     ╱‾‾‾╲
    │    ╱     ╲
    │   ╱       ╲
    └──┴───┴───┴──→ 주가
      OTM  ATM  ITM

ATM에서 가장 민감!
```

#### Theta (Θ) 자세히

**정의**: ∂V/∂t (시간에 대한 미분)

**의미**:
```
시간이 하루 지나면 옵션 가치는 얼마나 감소?

예:
  오늘: 옵션가 = 10원, Theta = -0.05
  내일: 옵션가 ≈ 10 - 0.05 = 9.95원
  모레: 옵션가 ≈ 9.95 - 0.05 = 9.90원
```

**시간 가치 붕괴 (Time Decay)**:
```
만기 1년 전: Theta 작음 (천천히 감소)
만기 1달 전: Theta 중간
만기 1주 전: Theta 큼 (빠르게 감소!)
만기 당일: 시간가치 = 0

  옵션 가치
    ↑
    │╲
    │ ╲
    │  ╲___
    │      ╲____
    └──────────╲──→ 시간
         만기 가까워질수록 가파름
```

**매도자 vs 매수자**:
```
옵션 매도자 (Seller):
  - Theta > 0 (유리!)
  - 시간 지날수록 이익
  - "시간이 친구"

옵션 매수자 (Buyer):
  - Theta < 0 (불리!)
  - 시간 지날수록 손해
  - "시간이 적"
```

#### 그릭스 간 관계

**Black-Scholes PDE와 연결**:
```
∂V/∂t + (r-q)S∂V/∂S + 0.5σ²S²∂²V/∂S² - rV = 0
  ↓        ↓            ↓
Theta    Delta        Gamma

Theta + 드리프트 + 변동성 효과 = 할인 효과
```

**실무에서의 사용**:
```
Delta: 방향성 헤징
Gamma: 재헤징 빈도
Theta: 시간 관리

예:
  Delta 헤지 → 중립 포지션
  Gamma 높음 → 자주 조정 필요
  Theta 음수 → 시간은 불리
```

# 문제 2: 최악자산 이진옵션

## 무엇을 계산하나?

**최악자산 이진옵션 (Worst-of Binary Option)**의 가격을 계산합니다.

### 최악자산 (Worst-of)

**두 개 주식 중 낮은 것 기준**:
```
삼성전자 (S1) = 110원
SK하이닉스 (S2) = 95원

min(S1, S2) = min(110, 95) = 95원 ← 이걸 사용!
```

### 이진옵션 (Binary/Digital)

**조건 만족 → 고정금액, 아니면 0**:
```
행사가 K = 100원

시나리오 1:
  S1 = 110, S2 = 105
  → min(110, 105) = 105 > 100
  → 페이오프 = 10,000원

시나리오 2:
  S1 = 110, S2 = 95
  → min(110, 95) = 95 < 100
  → 페이오프 = 0원
```

**일반 옵션과 비교**:
```
일반 콜옵션:
  S = 110, K = 100
  → 이익 = S - K = 10원 (연속적)

이진옵션:
  S > K → 10,000원
  S ≤ K → 0원 (불연속!)
```

## 왜 어려운가?

### 문제 1: 2차원

**두 주가를 동시에 추적**:
```
1차원 (주식 1개):
  그리드: [0, 50, 100, 150, 200]
  → 5개 점

2차원 (주식 2개):
  S2 ↑
  200 │ · · · · ·
  150 │ · · · · ·
  100 │ · · · · ·
   50 │ · · · · ·
    0 │ · · · · ·
    └─────────→ S1
      0  50 100 150 200

  → 5×5 = 25개 점!
  → 100×100이면 10,000개!
```

### 문제 2: 불연속성

**이진옵션은 계단함수**:
```
옵션 가격
  ↑
10000 │           ┌─────────
      │           │
      │           │ ← 갑자기 점프!
      │           │
    0 │───────────┘
      └───────────┼─────── 주가
                 100
```

**수치 계산 문제**:
- 급격한 변화 → 오차 증가
- 불안정성 발생

## 해결책 1: 오버헤지 (Overhedge)

### 행사가를 약간 낮춤

```python
원래 행사가: K = 100
오버헤지 파라미터: oh = 10
수정된 행사가: K_modified = 100 × (1 - 10/10000) = 99.9
```

### 왜 이렇게 하나?

**1. 완충지대 생성**:
```
원래 (K=100):
  99.99원 → 0원
  100.01원 → 10,000원  ← 급격한 변화!

수정 (K=99.9):
  99.89원 → 0원
  99.91원 → 10,000원  ← 행사가 근처 여유 생김
```

**2. 더 비싼 가격**:
```
만기 주가 = 99.95원

원래 K=100:
  99.95 < 100 → 0원

수정 K=99.9:
  99.95 > 99.9 → 10,000원!

→ 돈 받을 확률 ↑ = 옵션 가치 ↑ = 보수적 가격
```

**3. 실제 계약은 안 바뀜!**
```
고객과 계약: "행사가 100원"
내부 계산: "99.9원으로 계산"
결과: 약간 높은 가격 제시 (안전 마진)
```

### 트레이드오프

| 측면 | 오버헤지 없음 | 오버헤지 있음 |
|-----|------------|------------|
| **정확성** | ✅ 정확 | ❌ 약간 부정확 |
| **안정성** | ❌ 불안정 | ✅ 안정 |
| **계산** | 느림/오차 | 빠름/안정 |
| **가격** | 실제 | 보수적 (약간 높음) |

## 해결책 2: ADI/OSM (2차원 문제 해결)

### 2차원 문제가 왜 어려운가?

**행렬 크기 폭발**:
```
1차원: 100×100 행렬
2차원: 10,000×10,000 행렬!

→ 메모리: 100배 증가
→ 계산 시간: 1,000배 증가
```

### ADI (Alternating Direction Implicit)

**핵심 아이디어**: 2D를 두 개의 1D로 쪼개기

```
2차원 문제:
  10,000×10,000 행렬 1번 풀기
  → 매우 느림!

ADI 방법:
  100×100 행렬 200번 풀기
  → 훨씬 빠름!
```

#### 작동 방식

**반스텝 1: S1 방향만**
```
S2를 고정, S1만 계산:

S2=0일 때:   (S1=0, S2=0) → (S1=dx, S2=0) → ... (1D 문제)
S2=dy일 때:  (S1=0, S2=dy) → (S1=dx, S2=dy) → ... (1D 문제)
...
```

**반스텝 2: S2 방향만**
```
S1을 고정, S2만 계산:

S1=0일 때:   (S1=0, S2=0) → (S1=0, S2=dy) → ... (1D 문제)
S1=dx일 때:  (S1=dx, S2=0) → (S1=dx, S2=dy) → ... (1D 문제)
...
```

#### 시각화

```
전체 그리드:
  S2 ↑
  · · · · ·
  · · · · ·
  · · · · ·
  · · · · ·
  · · · · ·
  └─────────→ S1

반스텝 1 (S1 방향):
  S2 ↑
  → → → → →  각 행을 독립적으로 계산
  → → → → →
  → → → → →
  → → → → →
  → → → → →
  └─────────→ S1

반스텝 2 (S2 방향):
  S2 ↑
  ↑ ↑ ↑ ↑ ↑  각 열을 독립적으로 계산
  ↑ ↑ ↑ ↑ ↑
  ↑ ↑ ↑ ↑ ↑
  ↑ ↑ ↑ ↑ ↑
  ↑ ↑ ↑ ↑ ↑
  └─────────→ S1
```

### OSM (Operator Splitting Method)

**ADI와 비슷하지만 다른 분할**:

```
ADI:
  반스텝 1: S1 암시적, S2 명시적
  반스텝 2: S2 암시적, S1 명시적
  → 각 방향 반스텝

OSM:
  단계 1: S1 방향 풀스텝
  단계 2: S2 방향 풀스텝
  단계 3: 교차미분 보정
  → 각 방향 풀스텝 + 보정
```

### 교차미분 (Cross Derivative)

**두 자산의 상호작용**:
```
∂²V/∂S1∂S2

왜 필요?
→ 두 주식이 상관관계(correlation)를 가지기 때문
→ corr = 0.3 → 30% 같이 움직임

예시:
  삼성전자 ↑ → SK하이닉스도 ↑ (양의 상관관계)
  → 옵션 가치에 영향
```

## 2. 수치 방법

### 유한차분법 (FDM)

**PDE를 그리드로 이산화**:
```
연속 공간 → 이산 그리드
미분 → 차분 (difference)
```

### 시간 진행 방법

| 방법 | 특징 | 안정성 |
|-----|------|--------|
| **명시적** | 계산 쉬움 | 조건부 안정 |
| **암시적** | 방정식 풀어야 함 | 무조건 안정 |
| **Crank-Nicolson** | 둘의 평균 (θ=0.5) | 무조건 안정 + 2차 정확도 |

### 반복법

| 방법 | 설명 |
|-----|------|
| **Gauss-Seidel** | 기본 반복법 |
| **SOR** | 완화 파라미터(ω)로 가속 |
| **PSOR** | SOR + 제약조건 투영 |

#### Gauss-Seidel과 SOR 자세히

**Gauss-Seidel 방법**:
```
방정식: Ax = b
예: 5x + y = 10
    x + 3y = 8

반복:
  x_new = (10 - y_old) / 5
  y_new = (8 - x_new) / 3  ← 방금 계산한 x_new 사용!

수렴할 때까지 반복
```

**SOR (Successive Over-Relaxation)**:
```
Gauss-Seidel에 "가속"을 추가

x_temp = (10 - y_old) / 5        # Gauss-Seidel 계산
x_new = x_old + ω * (x_temp - x_old)  # ω로 조정
                └─ 변화량

ω = 1.0: 일반 Gauss-Seidel
ω > 1.0: 과완화 (Over-relaxation) → 더 빠르게 수렴
ω < 1.0: 저완화 (Under-relaxation) → 더 안정적

보통 ω = 1.2 사용 (20% 가속)
```

**왜 ω가 필요한가?**:
```
Gauss-Seidel:
  반복 1: x = 1.0
  반복 2: x = 1.5
  반복 3: x = 1.75
  반복 4: x = 1.875
  ...
  천천히 수렴

SOR (ω=1.2):
  반복 1: x = 1.0
  반복 2: x = 1.6  ← 더 빠름!
  반복 3: x = 1.92
  ...
  빨리 수렴
```

## 4. 실무적 고려사항

### 오버헤지


**위험 관리의 핵심**:

### 검증 방법

```

2. 서로 비교
3. 시장 가격과 대조
4. 민감도 분석 (파라미터 변화 테스트)
```

```
목표: 방정식 Ax = b 풀기
제약: x ≥ 내재가치

매 반복마다:
  1. 제약 무시하고 계산
  2. 제약 위반하면 → 제약 경계로 조정 (투영)
  3. 수렴할 때까지 반복
```

### ADI/OSM = 복잡한 문제 쪼개기

```
큰 문제: 서울→부산 (2차원 지도)

ADI 방식:
  1. 서울→대전 (위도만)
  2. 대전→부산 (경도만)
  → 각각은 1차원 문제!

OSM 방식:
  1. 서울→대전 (위도)
  2. 대전→대구 (경도)
  3. 대구→부산 (위도)
  4. 경로 보정
  → 각 구간은 간단!
```

### 오버헤지 = 안전 마진

```
건축 설계:
  실제 필요: 100m
  설계: 101m (안전 계수)

옵션 가격:
  실제 행사가: 100원
  계산: 99.9원 (보수적)
  → 약간 비싼 가격 제시
```
